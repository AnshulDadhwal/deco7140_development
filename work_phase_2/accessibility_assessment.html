Accessibility Assessment

Overview

This assessment focuses on selecting the WCAG 2.2 guidelines and success criteria most relevant to the Connect Communities Initiative website. The client brief highlights inclusivity, accessibility, and community trust, while user stories stress the importance of simple language, screen reader compatibility, low‑bandwidth access, and consistent navigation. Out of the 13 criteria studied, those selected here directly respond to client goals and user needs.

Perceivable

Guideline 1.1 Text Alternatives

This guideline is essential because accessibility is a client priority and users such as Ahmed depend on screen readers. Success Criterion 1.1.1 ensures all images, maps, and event posters include alt text so non‑text content can be understood.

Guideline 1.3 Adaptable

Since the website must support multiple devices and assistive technologies, content order must remain logical. Success Criterion 1.3.2 ensures the reading and navigation sequence is meaningful and consistent for users.

Guideline 1.4 Distinguishable

Clear visual presentation supports the client’s goal of reliable information access. Success Criterion 1.4.1 prevents reliance on color alone, Success Criterion 1.4.3 enforces minimum text contrast, and Success Criterion 1.4.4 allows text to be resized without losing functionality, directly helping users on mobile or older devices.

Operable

Guideline 2.1 Keyboard Accessible

Some users rely on keyboard navigation, making operability a key requirement. Success Criterion 2.1.1 ensures all interactive elements can be accessed without a mouse.

Guideline 2.4 Navigable

Predictable and clear navigation reduces user confusion and improves trust. Success Criterion 2.4.7 requires focus indicators that show which element is active, supporting both low‑vision and keyboard users.

Understandable

Guideline 3.1 Readable

Information must be easy to understand, especially for users with varying English proficiency. Success Criterion 3.1.5 ensures content is written at an accessible reading level, with translation options where needed.

Guideline 3.2 Predictable

Consistency in layout and interactions strengthens user confidence and saves time. Success Criterion 3.2.3 requires consistent navigation, and Success Criterion 3.2.4 ensures interface elements behave uniformly.

Guideline 3.3 Input Assistance

Input assistance supports accurate form submissions, which are vital for event postings and resource sharing. Success Criterion 3.3.1 makes errors clear, Success Criterion 3.3.2 provides labels and instructions, and Success Criterion 3.3.4 ensures error prevention and recovery to protect user input.

Conclusion

The chosen guidelines and success criteria not only ensure WCAG 2.2 compliance but also address real challenges identified in client goals and user stories. By focusing on perceivable, operable, and understandable principles, this website can provide an inclusive, trustworthy, and efficient experience for diverse international student communities.
